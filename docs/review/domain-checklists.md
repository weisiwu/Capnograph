# CapnoEasy 领域审核清单

<div class="doc-status" markdown>
<span>可执行清单</span><span>八个领域</span><span>按改动范围选择</span>
</div>

!!! tip "使用方法"
    先在[审核总览](review-guide.md)确定风险等级，再勾选本页相关领域。每个已勾选项都应能指向测试结果、回放样本、迁移报告或责任人结论。

## A. 临床数值、单位与报警

- [ ] 协议缩放、符号位、无效值和单位转换有真实帧样例；
- [ ] `mmHg`、`kPa`、`%` 下显示、阈值、量程、报告和设备下发一致；
- [ ] 低于下限、等于下限、区间内、等于上限、高于上限逐项测试；
- [ ] “检测到呼吸”前后的报警规则明确；
- [ ] 窒息、低电量、需校零、适配器异常的优先级符合批准需求；
- [ ] 报警启动、恢复、后台、页面切换和断连后的声音状态可解释；
- [ ] 配置失败或设备拒绝命令时，UI 不显示为成功。

## B. BLE、协议和设备控制

- [ ] Android 12+ 权限、旧版本兼容和权限拒绝路径通过；
- [ ] 扫描、连接、超时、断连、重连形成明确状态机；
- [ ] UUID、通知订阅、MTU/分包、校验和和命令序列有测试；
- [ ] 任务队列在超时、取消、断连和重试后不会重复下发危险命令；
- [ ] 校零、重置、关机有确认、反馈和失败处理；
- [ ] release 日志不输出完整原始患者或设备数据。

## C. 记录、波形和历史

- [ ] 未连接设备或患者字段不完整时不能开始；
- [ ] 快速双击、Activity 重建、后台和进程终止不会产生孤儿记录；
- [ ] chunk 写入成功后才移除内存数据，失败可重试且不重复；
- [ ] `chunkIndex` 连续且唯一，停止时剩余数据只写一次；
- [ ] `startTime <= sampleTimeMillis <= endTime`，跨分钟/跨日正确；
- [ ] 损坏块有明确错误提示或隔离策略；
- [ ] 患者与记录不会因偏好复用或并发操作串联。

## D. 数据库、备份与升级

- [ ] 实体变更同步提升 Room version、导出 schema 并提供 Migration；
- [ ] 使用上一发布版本的真实数据库执行升级测试；
- [ ] 备份包含主库/WAL/SHM 的一致快照；
- [ ] 恢复期间禁止新写入，重开后所有持有者获得新实例；
- [ ] 外键、唯一索引、Converter 和 GZIP/JSON 兼容旧数据；
- [ ] 失败时原数据仍可恢复，错误信息不包含患者信息。

## E. 隐私、安全与诊断

- [ ] 姓名、住院号、科室、床号、PDF、数据库和打印内容有数据清单；
- [ ] 偏好、应用私有目录、导出和系统备份的用途与保留期明确；
- [ ] 所有文件访问、后台位置和网络权限逐项证明必要性；
- [ ] Bugly/日志只上传错误阶段、类型、计数、布尔状态等最小字段；
- [ ] 导出由用户显式选择目标，失败或取消不残留副本；
- [ ] 仓库和产物不包含私钥、生产凭据或真实患者样本。

## F. PDF 与热敏打印

- [ ] 患者、医院、报告名称、时间、单位和生命体征来自同一记录快照；
- [ ] 波形切段、采样时间、坐标、正常范围带和统计值正确；
- [ ] 长记录分页不会丢段、重复段、裁切或造成内存峰值；
- [ ] 正式/调试水印和模板不会混淆；
- [ ] 文件名经过安全处理且避免暴露不必要的患者身份；
- [ ] 打印机断连、缺纸、重复点击和部分发送可恢复；
- [ ] PDF 与纸质输出的数据版本一致。

## G. Android/iOS 一致性

- [ ] 协议命令、单位、范围、默认值和状态位两端一致；
- [ ] 相同业务条件产生相同报警等级和记录结果；
- [ ] 平台特有限制写入需求、UI 和发布说明；
- [ ] 同一组设备录制/回放样本在两端得到一致关键值。

## H. 构建、测试和发布

- [ ] Android debug/release 均构建并验证关键流程；
- [ ] iOS scheme 在目标 Xcode/SDK 构建，签名材料不进入公共产物；
- [ ] `scripts/package.sh` 与 Docker 构建可复现；
- [ ] 产物记录版本、variant、提交 SHA、环境、哈希和签名；
- [ ] 真机覆盖目标系统、设备固件和打印机型号；
- [ ] 升级、全新安装、权限拒绝、离线、长记录和故障注入完成；
- [ ] 发布说明列出迁移、权限变化、平台差异和已知限制。
