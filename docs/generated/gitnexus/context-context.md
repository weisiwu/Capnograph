# context — context 模块

## 概述

`context` 模块是 CapnoEasy 项目的实体定位与上下文索引系统。它不包含任何可执行代码，而是通过一组 Markdown 文件为开发者（包括 AI 助手）提供从用户可见名称、代码实体、协议 ID、存储 ID 到源文件的快速映射能力。

该模块的核心价值在于：当开发者或 AI 需要理解、修改或扩展项目中的某个功能时，无需手动搜索整个代码库，而是通过本模块的索引文件直接定位到相关源文件和补充文档。

## 核心文件：entity-id-mapping.md

### 用途

这是项目最主要的定位索引文件。它将用户可能提到的各种名称（中文显示名、类名、枚举值、ID、UUID、表名、SharedPreferences key、资源名等）映射到对应的源文件路径和补充上下文文档。

### 使用方式

1. 将用户提到的显式名称与文件中的 `实体` 列匹配。
2. 读取匹配行里的 `源文件` 列，获取代码位置。
3. 仅在匹配行写有 `补充上下文` 时读取对应文档。文档和代码不一致时，以当前代码为准。
4. 如果改动新增、删除、重命名或移动了本文件列出的实体或 ID，需要同步更新本文件。

## 维护规则

1. **新增实体**：当添加新的类、枚举、常量、资源或功能时，需要在 `entity-id-mapping.md` 中添加对应行。
2. **删除/重命名实体**：同步更新索引文件中的对应行。
3. **移动源文件**：更新索引文件中的 `源文件` 列。
4. **补充文档**：当实体逻辑复杂时，在 `docs/` 下创建补充文档并在索引中引用。
5. **一致性**：文档和代码不一致时，以当前代码为准。
